-----

## WHAT REPLIT IS RESPONSIBLE FOR

Replit sets up the foundation that Cursor will build on top of:

1. Next.js 14+ project with App Router
1. Monorepo folder structure (not a flat project)
1. Core dependencies installed
1. PostgreSQL database created with all 18 tables + indexes
1. Database client configured (standard pg, no vendor lock-in)
1. .env file with DATABASE_URL connected + placeholder keys
1. Tailwind CSS initialized with QC brand tokens
1. A single test API route that confirms database connectivity
1. A bare homepage that loads (proves the app runs)

**What Replit does NOT do:** No pages, no UI, no business logic, no Stripe, no AI, no posting engine, no brand styling beyond the Tailwind config. That is all Cursor.

-----

## STEP 1: CREATE THE PROJECT

Create a new **Next.js** project on Replit. Use Next.js 14+ with the App Router (not Pages Router).

When Replit scaffolds it, it will create a flat project. You need to restructure it into a monorepo layout immediately.

## STEP 3: INSTALL DEPENDENCIES

In the dashboard app (`apps/dashboard/`), install:

```
# Core
next react react-dom typescript @types/react @types/node

# Styling
tailwindcss postcss autoprefixer
@tailwindcss/typography

# UI (initialize shadcn, don't add components yet, Cursor will)
npx shadcn@latest init

# Database (standard PostgreSQL client, NOT Supabase)
pg @types/pg

# Auth (install now, configure in Cursor)
next-auth @auth/core

# Utilities
clsx tailwind-merge lucide-react
```

In the workers app (`apps/workers/`), install:

```
typescript @types/node pg @types/pg
```

In the root, install:

```
turbo
```

**DO NOT install:** Supabase client libraries, Replit-specific database packages, or any vendor-specific database SDKs. Use the standard `pg` package only.

## STEP 4: SET UP REPLIT POSTGRESQL DATABASE

1. In your Replit project, enable the PostgreSQL database add-on
1. Replit will provide a `DATABASE_URL` connection string
1. Add it to your `.env` file

-----

## STEP 5: CREATE THE DATABASE CLIENT

Create `apps/dashboard/lib/db.ts`:

```typescript
import { Pool } from 'pg';

const pool = new Pool({
  connectionString: process.env.DATABASE_URL,
  ssl: process.env.NODE_ENV === 'production' ? { rejectUnauthorized: false } : false,
});

export default pool;

// Usage in API routes:
// import db from '@/lib/db';
// const result = await db.query('SELECT * FROM dealers WHERE id = $1', [id]);
```

This client works with Replit PostgreSQL now AND any other PostgreSQL provider later. The only thing that changes is the `DATABASE_URL` environment variable.

## STEP 6: RUN THE COMPLETE DATABASE MIGRATION

Create `migrations/001_full_schema.sql` with the entire schema below. Then run it against your Replit PostgreSQL database.

```sql
-- ============================================
-- QUANTUM CONNECT AI - COMPLETE DATABASE SCHEMA
-- 18 tables + indexes
-- Aligned with Doc 2 Final: per-rep model, admin panel, invitations, proxies, selectors
-- Run this ONCE on initial setup
-- ============================================

CREATE EXTENSION IF NOT EXISTS "pgcrypto";

-- ============================================
-- 1. DEALERS (billing entity / tenant)
-- Serves as the "account" for both dealerships and individual salespeople.
-- ============================================
CREATE TABLE dealers (
  id UUID PRIMARY KEY DEFAULT gen_random_uuid(),
  -- Account type: 'dealership' or 'individual'
  account_type TEXT NOT NULL DEFAULT 'dealership',
  name TEXT NOT NULL,
  slug TEXT UNIQUE NOT NULL,
  email TEXT NOT NULL,
  phone TEXT,
  address TEXT,
  city TEXT,
  state TEXT,
  zip TEXT,
  timezone TEXT NOT NULL DEFAULT 'America/New_York',
  logo_url TEXT,
  website_url TEXT,
  dms_provider TEXT,
  dms_feed_url TEXT,
  -- Stripe billing
  stripe_customer_id TEXT,
  stripe_subscription_id TEXT,
  stripe_subscription_status TEXT DEFAULT 'none',
  trial_ends_at TIMESTAMPTZ,
  -- Feature flags (SET ONLY BY STRIPE WEBHOOKS)
  feature_core_active BOOLEAN DEFAULT false,
  feature_custom_backgrounds BOOLEAN DEFAULT false,
  feature_visual_merchandising BOOLEAN DEFAULT false,
  feature_sales_dashboard BOOLEAN DEFAULT false,
  feature_sales_dashboard_setup_paid BOOLEAN DEFAULT false,
  -- Photo processing settings
  photo_background_mode TEXT DEFAULT 'replace',
  photo_background_template TEXT DEFAULT 'studio-white',
  photo_plate_blur BOOLEAN DEFAULT true,
  -- GHL config
  ghl_location_id TEXT,
  ghl_api_key_encrypted TEXT,
  ghl_webhook_url TEXT,
  ghl_pipeline_id TEXT,
  ghl_calendar_id TEXT,
  -- Limits (defaults, can be overridden per rep)
  max_daily_posts INTEGER DEFAULT 10,
  max_daily_group_posts INTEGER DEFAULT 3,
  -- Metadata
  subscription_plan TEXT DEFAULT 'core',
  created_at TIMESTAMPTZ DEFAULT NOW(),
  updated_at TIMESTAMPTZ DEFAULT NOW()
);

-- ============================================
-- 2. USERS (every person who can log in)
-- Roles: 'owner', 'admin', 'rep', 'superadmin'
-- owner = created the account, manages billing
-- admin = same as owner minus delete/transfer
-- rep = salesperson, sees only their own data
-- superadmin = QC internal team (admin panel access)
-- ============================================
CREATE TABLE users (
  id UUID PRIMARY KEY DEFAULT gen_random_uuid(),
  dealer_id UUID REFERENCES dealers(id) ON DELETE CASCADE,
  email TEXT UNIQUE NOT NULL,
  name TEXT NOT NULL,
  password_hash TEXT,
  role TEXT NOT NULL DEFAULT 'owner',
  avatar_url TEXT,
  is_active BOOLEAN DEFAULT true,
  last_login_at TIMESTAMPTZ,
  created_at TIMESTAMPTZ DEFAULT NOW(),
  updated_at TIMESTAMPTZ DEFAULT NOW()
);

-- ============================================
-- 3. INVITATIONS (rep invite tokens)
-- ============================================
CREATE TABLE invitations (
  id UUID PRIMARY KEY DEFAULT gen_random_uuid(),
  dealer_id UUID REFERENCES dealers(id) ON DELETE CASCADE,
  invited_by UUID REFERENCES users(id),
  email TEXT NOT NULL,
  name TEXT,
  token TEXT UNIQUE NOT NULL,
  role TEXT DEFAULT 'rep',
  status TEXT DEFAULT 'pending',  -- pending, accepted, expired, revoked
  expires_at TIMESTAMPTZ NOT NULL,
  accepted_at TIMESTAMPTZ,
  created_at TIMESTAMPTZ DEFAULT NOW()
);

-- ============================================
-- 4. PROXIES (managed by admin panel)
-- ============================================
CREATE TABLE proxies (
  id UUID PRIMARY KEY DEFAULT gen_random_uuid(),
  provider TEXT NOT NULL,           -- 'brightdata', 'decodo', etc.
  host TEXT NOT NULL,
  port INTEGER NOT NULL,
  username TEXT,
  password_encrypted TEXT,
  protocol TEXT DEFAULT 'http',     -- http, socks5
  geo_country TEXT,
  geo_state TEXT,
  geo_city TEXT,
  assigned_rep_id UUID,             -- FK added after sales_reps created
  status TEXT DEFAULT 'available',  -- available, assigned, failed, retired
  last_tested_at TIMESTAMPTZ,
  last_latency_ms INTEGER,
  created_at TIMESTAMPTZ DEFAULT NOW(),
  updated_at TIMESTAMPTZ DEFAULT NOW()
);

-- ============================================
-- 5. SALES REPS (each rep = one Facebook profile = one browser profile)
-- This is the core operational entity for posting.
-- ============================================
CREATE TABLE sales_reps (
  id UUID PRIMARY KEY DEFAULT gen_random_uuid(),
  dealer_id UUID REFERENCES dealers(id) ON DELETE CASCADE,
  user_id UUID REFERENCES users(id),
  name TEXT NOT NULL,
  email TEXT,
  phone TEXT,
  -- Facebook credentials (per rep)
  facebook_email TEXT,
  facebook_password_encrypted TEXT,
  facebook_2fa_secret TEXT,
  -- AdsPower browser profile (per rep)
  adspower_profile_id TEXT,
  -- Proxy assignment
  proxy_id UUID REFERENCES proxies(id),
  -- Operational status
  status TEXT DEFAULT 'pending',    -- pending, warming, active, paused, flagged, banned
  health_score INTEGER DEFAULT 100,
  ramp_day INTEGER DEFAULT 0,       -- 0-14, tracks 14-day ramp-up progress
  -- Daily posting tracking
  daily_post_limit INTEGER DEFAULT 10,
  posts_today INTEGER DEFAULT 0,
  daily_post_reset_at TIMESTAMPTZ,
  last_post_at TIMESTAMPTZ,
  last_warmup_at TIMESTAMPTZ,
  last_health_check_at TIMESTAMPTZ,
  -- Stats
  total_posts INTEGER DEFAULT 0,
  total_leads INTEGER DEFAULT 0,
  total_appointments INTEGER DEFAULT 0,
  total_flags INTEGER DEFAULT 0,
  -- Stripe (for Visual Merchandising per-seat billing)
  stripe_subscription_item_id TEXT,
  is_active BOOLEAN DEFAULT true,
  -- Vehicle assignment
  assigned_vehicle_ids UUID[],
  assignment_mode TEXT DEFAULT 'all', -- 'all', 'manual', 'round_robin'
  notes TEXT,
  created_at TIMESTAMPTZ DEFAULT NOW(),
  updated_at TIMESTAMPTZ DEFAULT NOW()
);

-- Add FK from proxies to sales_reps now that sales_reps exists
ALTER TABLE proxies ADD CONSTRAINT fk_proxy_rep
  FOREIGN KEY (assigned_rep_id) REFERENCES sales_reps(id) ON DELETE SET NULL;

-- ============================================
-- 6. SUBSCRIPTION ADD-ONS
-- ============================================
CREATE TABLE subscription_addons (
  id UUID PRIMARY KEY DEFAULT gen_random_uuid(),
  dealer_id UUID REFERENCES dealers(id) ON DELETE CASCADE,
  addon_type TEXT NOT NULL,

-- ============================================
CREATE TABLE stripe_events (

-- ============================================
CREATE TABLE vehicles (

  created_at TIMESTAMPTZ DEFAULT NOW()
);

-- ============================================

-- ============================================
CREATE TABLE photo_processing_log (

  notes TEXT,
  created_by UUID REFERENCES users(id),
  created_at TIMESTAMPTZ DEFAULT NOW(),
  updated_at TIMESTAMPTZ DEFAULT NOW()
);

-- ============================================
-- 18. SYSTEM ALERTS (admin alert inbox)
-- ============================================
CREATE TABLE system_alerts (
  id UUID PRIMARY KEY DEFAULT gen_random_uuid(),
  dealer_id UUID REFERENCES dealers(id),
  rep_id UUID REFERENCES sales_reps(id),
  alert_type TEXT NOT NULL,         -- checkpoint, ban, login_fail, post_fail, payment_fail, photo_fail
  severity TEXT DEFAULT 'warning',  -- info, warning, critical
  title TEXT NOT NULL,
  details TEXT,
  screenshot_url TEXT,
  is_resolved BOOLEAN DEFAULT false,
  resolved_by UUID REFERENCES users(id),
  resolved_at TIMESTAMPTZ,
  resolution_notes TEXT,
  created_at TIMESTAMPTZ DEFAULT NOW()
);
```

## STEP 8: CREATE THE DATABASE HEALTH CHECK API ROUTE

Create `apps/dashboard/app/api/health/route.ts`:

```typescript
import { NextResponse } from 'next/server';
import db from '@/lib/db';

export async function GET() {
  try {
    const result = await db.query('SELECT COUNT(*) FROM dealers');
    return NextResponse.json({
      status: 'ok',
      database: 'connected',
      dealers_count: result.rows[0].count,
      timestamp: new Date().toISOString(),
    });
  } catch (error) {
    return NextResponse.json({
      status: 'error',
      database: 'disconnected',
      error: String(error),
    }, { status: 500 });
  }
}
```
